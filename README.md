# 🎧 VoiceVocab — 声で答える英単語・熟語トレーニング

ランニング中などの手が使えない状況で、マイク付きイヤホン（AirPods Pro 2 等）を使って
音声だけで英単語・熟語の学習ができる Web アプリ（PWA）です。

## 機能

- **英→和 / 和→英** の出題形式
- **レベル指定**（Lv1・2 中学 / Lv3 高校 / Lv4 大学受験 / Lv5 上級）
  - 複数レベルをトグルで選択（デフォルト Lv3）。単語・熟語の切り替えも可能
- **ランク**: 直近50問の正答率とレベルから算出（0.1〜9.9、Lv5を全問正解で10）。
  設定画面に常時表示され、セッション終了時の結果画面でランクの変化（▲/▼）が確認できる
- **例文・類義語つき**: 英→和は例文の英文を出題時から表示（出題語を太字＆色つきで強調）。
  和訳・類義語・語源は答えそのもののヒントになるため、両方向とも回答後にのみ表示
  - 和→英では類義語での回答も正解として扱う（例: give up → quit / abandon でも正解）
  - 発音記号は設定でON/OFF切り替え（デフォルトOFF）
- **音声出題 + 音声回答**: 問題を読み上げ → マイクで回答（最大11秒） → 自動で正誤判定 → 次の問題へ
  - 無音でも「もう一度どうぞ」の再試行はせず、その分認識時間を長めに確保
  - 周囲のノイズで認識が途中で打ち切られても、待機時間内は自動で聞き直す（誤「聞き取れない」対策）
  - 正解はピンポーン♪、不正解はブザー音 + 正解の読み上げ + 5秒の確認待機
  - 英→和で正解したとき、音声認識された語以外の答え（別の訳語）も併記される
  - 「🖐 押し続けて一時停止」ボタン（または画面のどこでも押さえている間）は次の問題に進まない。
    指を離すと再開（例文をじっくり読みたいときに）
  - スキップは正誤判定なしで次の問題へ（同じセッション内で後ほど再出題）
  - 音声認識が5回連続で失敗すると全画面で一時中断し、「タッチすると続けます」と表示
    （マイクが塞がっている、周囲が騒がしい等に気づけるように）。画面タッチや回答表示中の
    発話（正解を声に出して練習した等）があった回は「反応あり」としてカウントしない
  - **同音異義語対策**: 音声認識は「げんしょう」と発話しても文脈次第で「現象」のような
    別の漢字に変換することがある。kuromoji.js（形態素解析）で認識結果と正解の両方を
    読み（ひらがな）に変換して比較することで、表記が違っても発音が合っていれば正解と判定する
- **品詞**: 問題の単語・熟語の直前にバッジ表示（名/動/形/副/熟）
- **語源分解**: 回答画面で単語を接頭辞・語根・接尾辞に分解し、問題文のすぐ下に
  それぞれの意味つきで表示
- **直近3問の履歴（間違えた問題のみ）**: 画面下部に固定表示され、スクロールしても常に見える
- **間違えた問題だけ再挑戦**: 結果画面のボタンから、いま間違えた問題のみを
  シャッフルして再出題できる
- **間違えた問題リスト**: 設定画面のボタンから、英→和／和→英を別セクションに分けて表示。
  「間違えた回数順」「最近順」の並び替えと、類義語・例文の表示ON/OFFを切り替え可能
- **正誤判定でレイアウトが動かない**: 回答後にしか見せない情報（和訳・類義語・語源など）も
  出題時点で最終的な内容までレンダリングしておき、visibilityの切り替えだけで見せる。
  DOMへの要素の出し入れをしないので、正誤表示の前後で例文やボタンの位置が一切ずれない
- **Galaxy Fold 等の折りたたみ対応**: 折りたたみ時（縦長画面）は縦いっぱいのレイアウト、
  展開時（広い画面）は2カラム表示・大きな文字に自動切り替え
- **忘却曲線に基づく間隔反復**（SM-2 簡易版）
  - 正解するごとに次回出題間隔が延びる: 1日 → 3日 → 7日 → 14日 → 30日 → 60日 → 120日
  - 間違えた問題は数問後にセッション内で再出題 + 10分後に復習期限
  - 復習期限が来た問題を最優先で出題
- **音声コマンド**: 「スキップ」「もう一度」「終了」（英語モードでは "skip" "repeat" "finish"）
- 画面スリープ防止（Wake Lock）
- 学習進捗はブラウザの localStorage に保存

## 同音異義語辞書について

`lib/kuromoji/` に形態素解析ライブラリ kuromoji.js と辞書データ(約17MB, gzip後約6MB)を同梱しています。
初回アクセス時にダウンロードされ、Service Workerがキャッシュするので2回目以降はオフラインでも動作します。
アプリの起動を妨げないよう、初回インストール時にはキャッシュせず、実際に使われた時点でキャッシュに保存されます。

## Androidでの使い方

音声認識にはマイク権限が必要で、**HTTPS でホストされたページ**でしか動きません
（PCのローカルテストは `localhost` でOK）。一番簡単なのは GitHub Pages です:

1. GitHub に新しいリポジトリを作成（例: `voicevocab`）
2. このフォルダのファイルを全部アップロード（Web画面から「Add file → Upload files」でOK）
3. リポジトリの Settings → Pages → Branch を `main` / `(root)` にして Save
4. 数分後に `https://<ユーザー名>.github.io/voicevocab/` で開ける
5. Android の **Chrome** で開き、マイクの使用を「許可」
6. メニュー →「ホーム画面に追加」でアプリとしてインストール（PWA）

### AirPods Pro 2 との組み合わせ

- Android とペアリングすればマイク・スピーカーとも自動で AirPods が使われます
- ポケットにスマホを入れたまま、読み上げを聞いて声で回答できます
- 「自動で次の問題へ進む」を ON にすれば完全ハンズフリー

## 注意事項

- 音声認識（Google のサーバーを利用）には**ネット接続が必要**です
- 判定は認識結果と正解リストの部分一致です。正しく答えたのに不正解になった場合は
  「⭕ 正解にする」ボタンで救済できます（進捗にも反映されます）
- iPhone の Safari では動作保証外（Android Chrome 推奨）

## 単語の追加

[words.js](words.js) の `WORDS` 配列に追記するだけです:

```js
{ en: "sustain", pos: "動", ipa: "/səˈsteɪn/", ja: ["維持", "持続", "支える"],
  ex: "We must sustain economic growth.", exJa: "経済成長を維持しなければならない。",
  syn: ["maintain", "support"], etym: [["sus-", "下から(sub-)"], ["tain", "保つ(tenereに由来)"]],
  lv: 4, k: "w" },  // lv: 1〜5, k: "w"=単語 "i"=熟語（pos: 名/動/形/副、熟語は"熟"。
                    // ipa/ex/exJa/syn/etymは省略可）
```

`ja` には音声認識が返しそうな表記（漢字・ひらがな）を複数入れておくと判定精度が上がります。
