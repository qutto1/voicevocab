# 🎧 VoiceVocab — 声で答える英単語・熟語トレーニング

ランニング中などの手が使えない状況で、マイク付きイヤホン（AirPods Pro 2 等）を使って
音声だけで英単語・熟語の学習ができる Web アプリ（PWA）です。

## 機能

- **英→和 / 和→英** の出題形式
- **レベル指定**（Lv1 中学 / Lv2 高校 / Lv3 大学受験 / Lv4 上級）
  - 複数レベルをトグルで選択（デフォルト Lv2）。単語・熟語の切り替えも可能
- **ランク**: 直近50問の正答率とレベルから算出（0.1〜9.9、最高レベルを全問正解で10）。
  英→和・和→英を独立に記録し、設定画面の右上に縦並びで常時表示（未開始は「--」、
  タップで算出方法をポップアップ表示）。結果画面ではランクの変化（▲/▼）と、
  過去10回分の推移グラフを出題方向ごとに表示する
- **例文・類義語つき**: 英→和は例文の英文を出題時から表示（出題語を太字＆色つきで強調）。
  和訳・類義語は答えそのもののヒントになるため、両方向とも回答後にのみ表示
  - 和→英では類義語での回答も正解として扱う（例: give up → quit / abandon でも正解）
  - 語形変化した回答も正解として扱う（例: acquire に対し gained / gaining、
    turn down に対し turned down）
  - 発音記号は設定でON/OFF切り替え（デフォルトOFF）
- **音声出題 + 音声回答**: 問題を読み上げ → マイクで回答 → 自動で正誤判定 → 次の問題へ
  - 回答の待ち時間は設定画面で選択（4〜10秒、デフォルト6秒）
  - 周囲のノイズで認識が途中で打ち切られても、待機時間内は自動で聞き直す（誤「聞き取れない」対策）
  - 発話は検知したが文字に解析できなかった場合は「もう一度どうぞ」と促し、同じ時間もう一度待つ
    （促すのは1回まで）
  - 「わからない」と答えると、待たずにすぐ不正解として正解を表示する
  - 正解は短いピンポン♪、不正解はブザー音 + 正解の読み上げ + 5秒の確認待機
  - 全問出題し終えると「問題は終了しました」と読み上げる（画面を見ていなくても気づけるように）
  - 英→和で正解したとき、音声認識された語以外の答え（別の訳語）も併記される
  - 「🖐 押し続けて一時停止」ボタン（または画面のどこでも押さえている間）は次の問題に進まない。
    指を離すと再開（例文をじっくり読みたいときに）
  - 「🤷 わからない」ボタン（または「わからない」と発話）で、待たずに不正解として正解を表示
  - 音声認識が5回連続で失敗すると全画面で一時中断し、「タッチすると続けます」と表示
    （マイクが塞がっている、周囲が騒がしい等に気づけるように）。画面タッチや回答表示中の
    発話（正解を声に出して練習した等）があった回は「反応あり」としてカウントしない
  - **同音異義語対策**: 音声認識は「げんしょう」と発話しても文脈次第で「現象」のような
    別の漢字に変換することがある。kuromoji.js（形態素解析）で認識結果と正解の両方を
    読み（ひらがな）に変換して比較することで、表記が違っても発音が合っていれば正解と判定する
- **品詞**: 問題の単語・熟語の直前にバッジ表示（名/動/形/副/熟）
- **語源分解**: 問題文のすぐ下に接頭辞・語根・接尾辞を表示。英→和では英語部分を出題時から
  表示し（英単語は問題として見えているため）、それぞれの意味だけを回答後に開示する
- **直近3問の履歴（間違えた問題のみ）**: 画面下部に固定表示され、スクロールしても常に見える
- **間違えた問題への挑戦**: 設定画面のスタート横の「❌ 間違えた問題」ボタンで、過去に間違えた
  問題だけをシャッフルして出題。結果画面の「間違いに再挑戦」でも同様に再出題できる。
  いずれもランクは変更しない
  - セッション内で間違えた問題は数問後に再出題されるが、そこで正解できた問題は
    「間違えた問題」から外れ、再挑戦の対象にならない
- **場面（passage）ベースの出題**: 問題は1語ずつ独立しているのではなく、「場面」から作られる。
  1つの場面＝会話またはニュースのまとまった文章で、その中に出てくる語が問題になる。
  出題は場面単位でまとめられ、指定レベルに合う語だけが問題になる（レベル外の語は文脈として読む）
  - **会話**: A/Bのやりとり形式の日常会話
  - **ニュース**: 実際の最新ニュースをもとに作成（毎日更新）。記事の文章はそのまま使わず、
    事実から学習者向けの英文を書き起こしている。出典と取得日をデータに保持
  - 設定画面の「問題タイプ」で会話・ニュースのどちらかを選ぶ
- **間違えた語が出てきた場面＋画像**: 結果画面の下部に、間違えた語を含む場面を丸ごと表示
  （間違えた語は下線で強調、各行に和訳つき）。もともと1つの場面として書かれた文章なので
  話が自然につながっている。その場面をもとに画像をWebから取得して併せて表示する
  （会話はイラスト、ニュースは実写。取得できない場合は文章のみ）
- **間違えた問題リスト**: 設定画面のボタンから、英→和／和→英を別セクションに分けて表示。
  「間違えた回数順」「最近順」の並び替えと、類義語・例文の表示ON/OFFを切り替え可能
- **正誤判定でレイアウトが動かない**: 回答後にしか見せない情報（和訳・類義語・語源など）も
  出題時点で最終的な内容までレンダリングしておき、visibilityの切り替えだけで見せる。
  DOMへの要素の出し入れをしないので、正誤表示の前後で例文やボタンの位置が一切ずれない
- **Galaxy Fold 等の折りたたみ対応**: 折りたたみ時（縦長画面）は縦いっぱいのレイアウト、
  展開時（広い画面）は2カラム表示・大きな文字に自動切り替え
- **忘却曲線に基づく間隔反復**（SM-2 簡易版）
  - 正解するごとに次回出題間隔が延びる: 1日 → 3日 → 7日 → 14日 → 30日 → 60日 → 120日
  - 間違えた問題は数問後にセッション内で再出題 + 10分後に復習期限
  - 復習期限が来た問題を最優先で出題
- **音声コマンド**: 「スキップ」「もう一度」「終了」「わからない」
  （英語モードでは "skip" "repeat" "finish" "I don't know"）
  - 正解と同じ語がコマンドと重なる場合（"give up" の正解「やめる」「quit」など）は
    コマンドとして扱わず、正解として判定する
- 画面スリープ防止（Wake Lock）
- 学習進捗・ランクはブラウザの localStorage に、各設定はスタート時に cookie に保存

## 同音異義語辞書について

`lib/kuromoji/` に形態素解析ライブラリ kuromoji.js と辞書データ(約17MB, gzip後約6MB)を同梱しています。
初回アクセス時にダウンロードされ、Service Workerがキャッシュするので2回目以降はオフラインでも動作します。
アプリの起動を妨げないよう、初回インストール時にはキャッシュせず、実際に使われた時点でキャッシュに保存されます。

## Androidでの使い方

音声認識にはマイク権限が必要で、**HTTPS でホストされたページ**でしか動きません
（PCのローカルテストは `localhost` でOK）。一番簡単なのは GitHub Pages です:

1. GitHub に新しいリポジトリを作成（例: `voicevocab`）
2. このフォルダのファイルを全部アップロード（Web画面から「Add file → Upload files」でOK）
3. リポジトリの Settings → Pages → Branch を `main` / `(root)` にして Save
4. 数分後に `https://<ユーザー名>.github.io/voicevocab/` で開ける
5. Android の **Chrome** で開き、マイクの使用を「許可」
6. メニュー →「ホーム画面に追加」でアプリとしてインストール（PWA）

### AirPods Pro 2 との組み合わせ

- Android とペアリングすればマイク・スピーカーとも自動で AirPods が使われます
- ポケットにスマホを入れたまま、読み上げを聞いて声で回答できます
- 「自動で次の問題へ進む」を ON にすれば完全ハンズフリー

## 注意事項

- 音声認識（Google のサーバーを利用）には**ネット接続が必要**です
- 結果画面の画像は外部サービス（image.pollinations.ai、取得できない場合は Openverse）から
  取得します。生成された英文が送信されるため、オフライン時やサービス停止時は表示されません
- 判定はやや寛容に作ってあります（類義語・語形変化・余計な語つきの回答・1文字違いの
  聞き間違いを正解として扱う）。それでも正しく答えたのに不正解になる場合は、音声認識が
  別の語を返している可能性が高いので、画面の「🎤」に出た認識結果を確認してください
- iPhone の Safari では動作保証外（Android Chrome 推奨）

## ファイル構成

| ファイル | 役割 |
|---|---|
| [index.html](index.html) | 画面構造（設定／学習／間違いリスト／結果） |
| [app.js](app.js) | 全ロジック（出題・音声認識/合成・判定・SRS・ランク・復習表示） |
| [words.js](words.js) | **語彙辞書**。語の意味・品詞・発音記号・類義語・語源を持つ（191語） |
| [passages.js](passages.js) | **場面データ**。会話・ニュースの文章と、そこに出てくる語の対応（43場面） |
| [style.css](style.css) | 全スタイル |
| [sw.js](sw.js) | Service Worker（オフラインキャッシュ） |
| `lib/kuromoji/` | 形態素解析ライブラリと辞書（唯一の外部依存） |

出題される語は `passages.js` の場面に登場する語だけです。`words.js` に載っていても
どの場面にも出てこない語は出題されません。

## 問題の追加

### 1. 場面を追加する（[passages.js](passages.js)）

```js
{
  id: "t025", type: "talk", title: "カフェで注文する",
  lines: [
    { sp: "A", en: "I'd like to order a sandwich.", ja: "サンドイッチを注文したいのですが。" },
    { sp: "B", en: "Sure. Would you like something to drink?", ja: "かしこまりました。お飲み物はいかがですか。" },
  ],
  targets: [
    { en: "order", k: "w", line: 0 },   // line = その語が出てくる lines の番号
  ],
},
```

- `type`: `"talk"`=会話（`lines` に話者 `sp` を付ける）／ `"news"`=ニュース（地の文）
- ニュースは `source`（出典URL）と `fetchedAt`（取得日）も持たせる。記事の文章を
  そのまま使わず、事実をもとに学習者向けの英文を書き起こすこと（著作権のため）
- **同じ語を同じタイプの中で重複させない**（会話とニュースにまたがるのはOK）
- **`targets` の語は、その行の英文に実際に出てくること**。活用形（`turned down` ↔
  `turn down`）は自動で照合されるが、出てこない語を指定すると下線が引かれない

### 2. 辞書に語を追加する（[words.js](words.js)）

場面で使う語が辞書にない場合は追記します:

```js
{ en: "sustain", pos: "動", ipa: "/səˈsteɪn/", ja: ["維持", "持続", "支える"],
  ex: "We must sustain economic growth.", exJa: "経済成長を維持しなければならない。",
  syn: ["maintain", "support"], etym: [["sus-", "下から(sub-)"], ["tain", "保つ(tenereに由来)"]],
  lv: 4, k: "w" },  // lv: 1,2=中学 3=高校 4=大学受験 5=上級 / k: "w"=単語 "i"=熟語
                    // pos: 名/動/形/副、熟語は"熟"。ipa/ex/exJa/syn/etymは省略可
```

`ja` には音声認識が返しそうな表記（漢字・ひらがな）を複数入れておくと判定精度が上がります。
`ex`/`exJa` は場面に属さない語の予備であり、出題時の例文は場面の行が使われます。

### 3. 検証する

追加後は、語が実際に文中に出てくるか・辞書にあるか・重複がないかを確認します
（照合はアプリ本体と同じロジックで行うこと）。レベル別の語数もここで確認できます。

## ニュースの自動更新

毎朝6時に、最新ニュースから `type:"news"` の場面を作って差し替えるスケジュールタスクが
動きます（タスク定義: `~/.claude/scheduled-tasks/voicevocab-news-update/`）。
アプリを閉じているときに時刻が来た場合は、次回起動時に実行されます。
